# Day 7 - Claude Usage Strategy

## 🎯 Objective
Generate a personalized Claude usage strategy based on my actual tasks as an ECE student and document the workflow.

---

## 🤖 My Personalized Task Workflow

| Task | Best Model | Effort | Reason |
|------|-----------|--------|--------|
| Understanding op-amp / amplifier theory | Sonnet | Standard | Needs depth but not Opus-level; Sonnet explains derivations well |
| Debugging C / Python lab code (OPPE) | Sonnet | High | Bugs require careful reasoning; high effort catches subtle errors |
| AWK scripting / Linux practicals | Sonnet | Standard | Routine scripting; standard effort is plenty |
| AKTU exam MCQ practice | Haiku | Low | Fast recall; Haiku handles MCQs at speed |
| 7-mark AKTU answer writing | Sonnet | High | Structured derivations + diagrams need careful generation |
| PCB component selection & layout | Sonnet | High | Trade-offs (thermal, signal, cost) need thorough analysis |
| Full PCB system architecture | Opus | Max | Multi-domain design needs Opus's deep reasoning |
| Learning new ECE concepts (LVDT, ADC) | Sonnet | Standard | Explanations + analogies; standard effort sufficient |
| Quick formula / unit check | Haiku | Low | Zero reasoning needed; Haiku is fastest |
| Research / literature summary | Opus | High | Synthesis across sources needs Opus; Max usually overkill |

---

## 🧠 Key Learnings

- **Use Haiku** for fast, repetitive tasks like MCQ practice and quick formula checks — no need to waste Sonnet/Opus on simple lookups.
- **Use Sonnet** for the majority of engineering and coding tasks — it hits the sweet spot of depth and speed.
- **Save Opus** only for complex, multi-domain problems like full PCB system architecture or deep research synthesis.
- Matching the model to the task saves time and improves output quality.
- Claude works best when you give it structured, specific prompts — vague questions give vague answers.
